# UI設計（P1-01〜P1-03）

この文書は、実装前タスク `P1-01: 画面構成設計（単画面）`  
`P1-02: UI挙動設計`、`P1-03: UI表示ルール定義` の成果物です。  
初期スコープ（`SELECT` / `FROM` / `WHERE`）に対応するための最小UI構成と挙動を定義します。

## 1. 画面目的

- SQLを1文入力し、説明生成を実行し、結果とメッセージを同一画面で確認できるようにする。
- 学習用途を想定し、操作手順を「入力 → 実行 → 結果確認」の3段階に限定する。

## 2. 対象画面

- 画面数: 1（単画面）
- 想定技術: WPF（WinFormsでも同等構成を再現可能）
- 想定ウィンドウタイトル: `SQL説明ツール`

## 3. 主要UI要素

P1-01で必須とされた4要素を以下で定義する。

1. 入力欄（SQL入力テキストエリア）
   - 複数行入力を許可する。
   - 1回の説明対象は単一SQL文とする（スコープ準拠）。

2. 実行ボタン
   - ラベル案: `説明する`
   - クリック時に入力欄の内容を解析処理へ渡す。

3. 説明表示欄
   - 生成された説明文（要約 + 句別説明）を表示する。
   - 初期表示は空、またはプレースホルダー文とする。

4. メッセージ欄
   - 空入力、未対応構文、解析不能などの状態メッセージを表示する。
   - 正常時は非表示、または空表示とする。

## 4. 画面配置（レイアウト）

視線の流れを「上から下」に統一し、学習コストを下げる。

- 上段: 入力欄
- 中段: 実行ボタン
- 下段: 説明表示欄
- 最下段: メッセージ欄

### 4-1. 簡易ワイヤーフレーム

```text
+--------------------------------------------------+
| SQL入力                                           |
| +----------------------------------------------+ |
| | SELECT id, name                              | |
| | FROM users                                   | |
| | WHERE active = 1;                            | |
| +----------------------------------------------+ |
|                                                  |
| [ 説明する ]                                     |
|                                                  |
| 説明結果                                         |
| +----------------------------------------------+ |
| | 要約: ...                                     | |
| | - SELECT: ...                                 | |
| | - FROM: ...                                   | |
| | - WHERE: ...                                  | |
| +----------------------------------------------+ |
|                                                  |
| メッセージ: （未対応構文があります）             |
+--------------------------------------------------+
```

## 5. 要素ごとの配置方針

- 入力欄
  - 画面内で最も広い領域を確保する（ユーザー入力を優先）。
  - 折り返し表示を有効にし、横スクロール依存を避ける。

- 実行ボタン
  - 入力欄直下に単独配置し、操作対象を明確化する。
  - 初期実装では追加ボタン（クリア等）を置かない。

- 説明表示欄
  - 入力欄に次ぐ広さを確保し、複数行表示を前提とする。
  - 編集不可（読み取り専用）を基本とする。

- メッセージ欄
  - 説明表示欄の下に固定し、状態通知の場所を一貫化する。
  - 1〜2行で収まる短文表示を基本とする。

## 6. UI挙動設計（P1-02）

主要状態ごとの挙動を以下に定義する。

### 6-1. 実行時（正常）

- 条件
  - 入力欄が空でない。
  - SQLが初期対応範囲（`SELECT` / `FROM` / `WHERE`）で解釈可能。
- 挙動
  1. 実行ボタン押下で説明生成処理を開始する。
  2. 説明表示欄を最新結果で上書きする。
  3. メッセージ欄は空表示にする（または非表示）。
- 表示内容
  - 説明表示欄: 要約 + 句別説明。
  - メッセージ欄: 何も表示しない。

### 6-2. 空入力時

- 条件
  - 入力欄が空文字、または空白/改行のみ。
- 挙動
  1. 説明生成処理は実行しない。
  2. 既存の説明表示欄の内容は保持する（ユーザーの参照を維持）。
  3. メッセージ欄に入力促進メッセージを表示する。
- 表示内容（文言例）
  - `SQLを入力してから「説明する」を押してください。`

### 6-3. 未対応構文時

- 条件
  - SQLとしては解釈可能だが、初期対応外の要素（例: `JOIN`、副問い合わせ）を含む。
- 挙動
  1. 説明生成処理は中断する（部分説明は行わない）。
  2. 説明表示欄は更新しない（直前の正常結果を保持）。
  3. メッセージ欄に未対応であることと対象要素を表示する。
- 表示内容（文言例）
  - `未対応の構文を検出しました: JOIN`

### 6-4. 解析不能時

- 条件
  - 構文崩れなどによりSQLとして解釈できない。
- 挙動
  1. 説明生成処理は失敗として扱う。
  2. 説明表示欄は更新しない（直前の正常結果を保持）。
  3. メッセージ欄に再入力を促すエラーメッセージを表示する。
- 表示内容（文言例）
  - `SQLを解析できませんでした。構文を確認してください。`

## 7. UI表示ルール定義（P1-03）

初期表示、テキスト整形、長文入力時の取り扱いを以下に定義する。

### 7-1. 初期表示

- 入力欄
  - 空の状態で開始する。
- 説明表示欄
  - 空表示、または案内文 `ここに説明結果が表示されます。` を表示する。
- メッセージ欄
  - 空表示（エラー状態ではないため）。

### 7-2. 改行 / 箇条書きルール

- 説明表示欄の出力フォーマット
  1. 1行目: 要約（`要約: ...`）
  2. 2行目以降: 句別説明を箇条書き
     - `- SELECT: ...`
     - `- FROM: ...`
     - `- WHERE: ...`
- 改行方針
  - 1要素1行を原則とし、可読性を優先する。
  - 句が存在しない場合はその行を出力しない（空行は増やさない）。

### 7-3. 長文入力時の扱い

- 入力欄
  - 垂直スクロールを有効にする。
  - 折り返し表示を有効にし、横スクロール依存を避ける。
  - 入力長の目安超過時（例: 2000文字超）に警告表示を行う実装拡張余地を残す。
- 説明表示欄
  - 読み取り専用 + 垂直スクロールを前提とする。
  - 長文でも表示崩れしないよう、固定幅フォントは必須としない（通常UIフォントで可）。

### 7-4. メッセージ表示ルール

- 同時に複数メッセージは表示しない（最新1件のみ）。
- メッセージ優先度は `解析不能 > 未対応構文 > 空入力 > なし` とする。
- メッセージ文は短文で具体的にし、次の行動が分かる内容にする。
