# UI設計（P1-01）

この文書は、実装前タスク `P1-01: 画面構成設計（単画面）` の成果物です。  
初期スコープ（`SELECT` / `FROM` / `WHERE`）に対応するための最小UI構成を定義します。

## 1. 画面目的

- SQLを1文入力し、説明生成を実行し、結果とメッセージを同一画面で確認できるようにする。
- 学習用途を想定し、操作手順を「入力 → 実行 → 結果確認」の3段階に限定する。

## 2. 対象画面

- 画面数: 1（単画面）
- 想定技術: WPF（WinFormsでも同等構成を再現可能）
- 想定ウィンドウタイトル: `SQL説明ツール`

## 3. 主要UI要素

P1-01で必須とされた4要素を以下で定義する。

1. 入力欄（SQL入力テキストエリア）
   - 複数行入力を許可する。
   - 1回の説明対象は単一SQL文とする（スコープ準拠）。

2. 実行ボタン
   - ラベル案: `説明する`
   - クリック時に入力欄の内容を解析処理へ渡す。

3. 説明表示欄
   - 生成された説明文（要約 + 句別説明）を表示する。
   - 初期表示は空、またはプレースホルダー文とする。

4. メッセージ欄
   - 空入力、未対応構文、解析不能などの状態メッセージを表示する。
   - 正常時は非表示、または空表示とする。

## 4. 画面配置（レイアウト）

視線の流れを「上から下」に統一し、学習コストを下げる。

- 上段: 入力欄
- 中段: 実行ボタン
- 下段: 説明表示欄
- 最下段: メッセージ欄

### 4-1. 簡易ワイヤーフレーム

```text
+--------------------------------------------------+
| SQL入力                                           |
| +----------------------------------------------+ |
| | SELECT id, name                              | |
| | FROM users                                   | |
| | WHERE active = 1;                            | |
| +----------------------------------------------+ |
|                                                  |
| [ 説明する ]                                     |
|                                                  |
| 説明結果                                         |
| +----------------------------------------------+ |
| | 要約: ...                                     | |
| | - SELECT: ...                                 | |
| | - FROM: ...                                   | |
| | - WHERE: ...                                  | |
| +----------------------------------------------+ |
|                                                  |
| メッセージ: （未対応構文があります）             |
+--------------------------------------------------+
```

## 5. 要素ごとの配置方針

- 入力欄
  - 画面内で最も広い領域を確保する（ユーザー入力を優先）。
  - 折り返し表示を有効にし、横スクロール依存を避ける。

- 実行ボタン
  - 入力欄直下に単独配置し、操作対象を明確化する。
  - 初期実装では追加ボタン（クリア等）を置かない。

- 説明表示欄
  - 入力欄に次ぐ広さを確保し、複数行表示を前提とする。
  - 編集不可（読み取り専用）を基本とする。

- メッセージ欄
  - 説明表示欄の下に固定し、状態通知の場所を一貫化する。
  - 1〜2行で収まる短文表示を基本とする。
