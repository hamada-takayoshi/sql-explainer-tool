# 非機能要件（最小合意）

本ドキュメントは、SQL説明ツール（MVP）の実装前に合意すべき最小の非機能要件を定義する。  
対象タスク: P5-03

## 1. 対応OS

- 対応対象: Windows 10 / Windows 11
- 前提理由:
  - 本MVPは .NET デスクトップアプリ（WPF/WinForms想定）として実装するため
  - 初期フェーズでは利用環境をWindowsに限定し、検証と保守コストを抑える
- 判定基準:
  - 上記OSでアプリ起動および基本動作（正常/空入力/未対応）が確認できること

## 2. レスポンス目標

- 対象処理: SQL入力〜説明文表示まで（ローカル実行）
- 目標値:
  - 通常入力（1〜500文字程度）で **1秒以内** に応答する
- 許容範囲:
  - 開発機性能差を考慮し、最大でも **2秒以内** を許容上限とする
- 判定基準:
  - 手動確認で体感遅延がなく、計測時に上記目標内であること

## 3. 入力長の目安

- 想定入力長: 1〜1000文字
- MVPでの運用方針:
  - 1〜1000文字はサポート対象
  - 1001文字以上は「長文入力として注意喚起メッセージを表示」し、動作保証外（ベストエフォート）とする
- 判定基準:
  - 1000文字以内で異常終了しない
  - 長文時の扱い（注意喚起）がUI仕様と矛盾しない

## 4. 運用上の補足

- 本定義はMVP用の最小合意であり、将来フェーズで必要に応じて改定する
- 性能の厳密ベンチマークや多OS対応（macOS/Linux）は本フェーズの対象外とする
