## 実装時の注意点

- UTはロジック層を中心に作成し、UIとロジックを分離したまま進める。
- 1テスト1責務を基本とし、失敗時に原因が特定しやすい構成にする。
- 未対応構文は明確なメッセージを返し、仕様としてドキュメントに残す。
- 常にビルド可能な状態を維持し、小さな単位で進める。
