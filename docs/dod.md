# DoD（Definition of Done）

本ドキュメントは、SQL説明ツールの実装完了判定を主観なく運用するための基準を定義する。

## 1. 判定対象

- 対象: MVP（SELECT / FROM / WHERE 対応）の実装完了時
- 判定単位: リリース候補ブランチまたはマージ前の最終コミット

## 2. 完了条件

以下の **全項目を満たした場合のみ**「完了」と判定する。

### 2.1 Build 成功

- .NET プロジェクトがエラーなくビルドできること
- 警告は許容するが、対応方針（許容理由または修正予定）を記録すること

判定方法（例）:

- `dotnet build` が終了コード 0 を返す

### 2.2 基本動作確認

対象機能（MVP）に対して、少なくとも以下が確認できること。

- 正常系: SELECT / FROM / WHERE を含む入力で説明文が表示される
- 空入力: 実行時にエラーメッセージ（入力促し）が表示される
- 未対応入力: JOIN など未対応要素を含む場合に、未対応である旨のメッセージが表示される

判定方法:

- 定義済み手動確認シナリオを実施し、期待結果と一致する

### 2.3 README 更新

- 利用者向け README に、少なくとも以下が記載されていること
  - ツール概要
  - 実行手順（ビルド/起動）
  - 対応SQL範囲（MVP）
  - 既知の制約（未対応句など）

判定方法:

- README の該当セクションが存在し、現行実装と不整合がない

## 3. 判定フロー

1. `dotnet build` 実行結果を確認
2. 手動確認シナリオ結果を確認
3. README 記載内容を確認
4. 3項目すべて満たす場合のみ完了判定

## 4. 判定記録フォーマット

運用時は以下テンプレートで記録する。

- Build: Pass / Fail（ログ参照先）
- 基本動作: Pass / Fail（シナリオ結果参照先）
- README: Pass / Fail（確認コミット）
- 最終判定: Done / Not Done

## 5. 除外事項

- 非機能要件（応答時間、対応OSなど）の合否判定は本DoDでは扱わない
- これらは別紙（non-functional.md）で定義し、P5-03で合意する
