# スコープ定義（P0-01）

この文書は、実装前タスク `P0-01` の成果物です。  
SQL説明ツールの初期実装で扱うスコープ候補を列挙し、確認順に沿って確定内容を整理します。

## 1. スコープ候補の羅列

### 1-1. 機能スコープ候補
- SQL入力（複数行テキスト入力）
- 「説明する」実行操作
- 説明結果の表示
- 簡易解析（句の抽出）
- エラー/警告メッセージ表示
- 入力クリア操作（候補）
- SQL整形表示（候補）
- キーワード強調表示（候補）

### 1-2. 対応SQL候補
- `SELECT`
- `FROM`
- `WHERE`
- `ORDER BY`（候補）
- `GROUP BY`（候補）
- `JOIN`（候補）
- サブクエリ（候補）

### 1-3. 非機能スコープ候補
- Windowsで動作
- ローカル実行のみ（ネットワーク不要）
- 依存追加を最小化
- 応答時間目標（候補）

## 2. 確認順と確定内容

以下の順番で確認し、初期実装のスコープを確定する。

### 確認1: 目的と利用シーン
- 確認内容: 学習用途として、SQLを「人が理解しやすい文章」にすることを主目的とする。
- 確定: 初期実装は学習支援に必要な最小機能を優先する。

### 確認2: 初期機能の境界
- 確認内容: 入力・実行・説明表示・最低限のメッセージ表示を対象とする。
- 確定（対応）:
  - SQL入力
  - 実行操作
  - 説明表示
  - 空入力/解析不能時のメッセージ表示
- 確定（対象外）:
  - 入力クリア専用ボタン（初期は必須にしない）
  - SQL整形
  - キーワード強調

### 確認3: 対応SQL句の境界
- 確認内容: 初期は基本句のみ対応し、複雑構文は対象外とする。
- 確定（対応）:
  - `SELECT`
  - `FROM`
  - `WHERE`
- 確定（対象外）:
  - `ORDER BY`
  - `GROUP BY`
  - `JOIN`
  - サブクエリ

### 確認4: 入力前提
- 確認内容: どの形式のSQL入力を受け付けるか。
- 確定:
  - 1回の説明対象は単一SQL文
  - コメントや方言差は初期では厳密対応しない
  - 著しく複雑なSQLは「未対応」案内を返す

### 確認5: 非機能・制約
- 確認内容: 開発方針に沿った最小制約を置く。
- 確定:
  - デスクトップ向け（WPF/WinForms想定）
  - 依存関係は最小限
  - 常にビルド可能な状態を維持

## 3. 初期実装スコープ（最終確定）

### 3-1. 対応範囲（In Scope）
1. SQL入力（テキスト）
2. 実行操作（説明生成トリガー）
3. `SELECT` / `FROM` / `WHERE` の簡易説明
4. 空入力・解析不能・未対応構文のメッセージ表示

### 3-2. 対象外（Out of Scope）
1. `JOIN`、サブクエリ、`GROUP BY`、`ORDER BY` の詳細説明
2. SQL整形、ハイライト表示
3. 高度な構文検証・完全パーサー相当の解析

### 3-3. 判断ルール
- 判断に迷う要件は「学習用途の最小機能に必要か」で決める。
- 必要性が低いものは初期スコープから外し、拡張候補として記録する。
